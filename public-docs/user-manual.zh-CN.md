# PKV Sync 用户手册

[English](./user-manual.md) | 简体中文 | [繁體中文](./user-manual.zh-Hant.md) | [日本語](./user-manual.ja.md) | [한국어](./user-manual.ko.md)

文档版本：v1.2.1。

本手册面向连接到已有 PKV Sync 服务器的 Obsidian 用户。开始之前，请向服务器管理员索取服务器分享 URL，以及一个账号或邀请码。

## 手动安装插件

1. 从对应的 GitHub Release 下载 `pkv-sync-plugin.zip`。
2. 将压缩包解压到你的笔记库：

   ```text
   <vault>/.obsidian/plugins/pkv-sync/
   ```

3. 在 Obsidian 中启用社区插件。
4. 启用 **PKV Sync**。

解压后的目录中应包含 `main.js`、`manifest.json` 与 `styles.css`。

## 插件更新

PKV Sync 设置页面包含一个 **更新** 区域。默认情况下，插件会向已连接的 PKV Sync 服务器查询其内置的插件版本；对于自托管部署来说，这是首选的更新来源，因为升级服务器时会同步发布配套的插件资源。当服务器配置了 `public_host` 时，插件资源 URL 仍会指向该对外主机。如有需要，你可以将更新来源切换为 GitHub Releases。

当有可用更新时，**立即更新** 会下载 `main.js`、`manifest.json`，以及（若存在）`styles.css`，校验 SHA-256 哈希后写入插件文件，并提示你重新加载 Obsidian。命令面板中也提供 **PKV Sync：检查 PKV Sync 插件更新**。

## 连接到服务器

服务器分享 URL 通常形如：

```text
https://sync.example.com/k_xxx/
```

打开 **设置 -> PKV Sync**，粘贴分享 URL，然后点击 **连接**。如果部署密钥已包含在 URL 中，插件会自动填入。

如果你输错了服务器，或者需要切换到另一台自托管服务器，可以在登录界面使用 **修改服务器**，直接回到服务器设置，无需重新安装插件。

## 登录或注册

注册行为取决于服务器的运行时设置：

- **关闭**：必须由管理员为你创建账号。
- **仅限邀请**：输入管理员提供的邀请码。
- **开放**：可以直接创建账号。

公开注册和邀请码注册都使用与 setup 相同的强密码策略：至少 12 个字符，并包含大写字母、小写字母和数字。

登录后，选择一个已有的远端笔记库或新建一个。当你连接的本地笔记库与所选远端笔记库已经完全一致时，PKV Sync 会将匹配的文件直接纳入本地同步索引，而不会为整个笔记库生成一整套冲突文件。

## 同步行为

PKV Sync 运行在 Obsidian 中，同步当前笔记库：

- 本地文件改动会在短暂去抖间隔后推送。
- 远端改动会被周期性轮询。
- 在设置页面和命令面板中都可以执行手动同步。
- 相关的文件创建/修改/删除事件会触发一次同步。
- 窗口失焦也可能触发同步。
- 启动时，会根据笔记库内容和本地同步索引检测出尚未同步的本地改动。

上传大型附件期间请保持 Obsidian 打开。插件在连接成功后会读取服务器配置，并使用服务器提供的文本扩展名列表与最大文件大小规则。

## 选择性同步 `.obsidian`

PKV Sync 可以通过每个笔记库独立的允许列表来同步选定的 Obsidian 配置文件。新建的远端笔记库会预置主题、CSS 片段、快捷键、应用偏好、外观偏好以及已启用插件列表的规则。

已有的远端笔记库会保持空允许列表，直到你主动启用。在 **设置 -> PKV Sync** 中选中当前笔记库，编辑 **.obsidian 同步允许列表**，然后保存。推荐起始列表按钮会填入与新建笔记库相同的起始规则。

默认不会同步插件代码和插件设置。在添加 `.obsidian/plugins/**` 或插件 `data.json` 之类的高级规则前，请先阅读 [`dot-obsidian-sync-howto.md`](./dot-obsidian-sync-howto.md)。

## 上次同步时间

设置页面会以相对时间显示最近一次成功的同步。使用旁边的小箭头可以展开显示精确时间戳，格式为：

```text
YYYY/MM/DD HH:MM:SS
```

插件使用所选 IANA 时区，默认为 `Asia/Shanghai`。

## 外观

插件设置页面提供一个 **外观** 下拉，包含三个选项：

- **自动**（默认）：跟随当前 Obsidian 应用主题。
- **浅色** / **深色**：无论应用主题如何，都为 PKV Sync 设置界面强制使用指定主题。

此设置只影响插件自身的弹窗（设置、同步状态、冲突解决器、文件/仓库历史），不会改变 Obsidian 应用本身的主题。

## 历史、差异与恢复

当服务器报告支持历史记录，且插件设置中 **开启历史与差异界面** 处于打开状态时，你可以从以下入口查看文件历史：

- **PKV Sync：查看文件历史**
- **PKV Sync：查看仓库历史**
- 文件右键菜单中的 **PKV Sync：文件历史**
- 文件右键菜单中的 **PKV Sync：与上一版对比**

文件历史弹窗列出当前文件的提交记录，包含时间、设备、commit id 和变更类型。文本文件可以显示统一格式 diff。二进制文件可以列出并恢复，但 PKV Sync 不会渲染二进制 diff。

被当前同步过滤规则拒绝的路径会从文件历史、仓库历史摘要、diff 视图和文件读取界面中隐藏。

仓库历史弹窗列出整个笔记库最近的提交记录，包含时间、设备、变更数量和 commit id。打开某个 commit 可以查看其按文件的变更摘要。

恢复某个版本时，PKV Sync 会从服务器读取所选历史版本的内容，将其写回本地 Obsidian 笔记库，然后由正常的同步引擎将这次写入作为新的 commit 推送。如果当前本地文件与上次同步的哈希不一致，确认对话框会警告未同步的本地修改将被覆盖。

PKV Sync 不会在插件中保存完整的离线历史缓存。历史与差异视图都需要服务器可达。

## 冲突文件

如果两台设备离线编辑了同一个文件，PKV Sync 会保留两个版本。远端或本地的备用版本会被保存为生成的冲突文件：

```text
note.md
note.conflict-2026-04-25-143022-Desktop.md
```

生成的冲突文件会被排除在后续同步之外。请在 Obsidian 中查看两份文件，合并你想保留的内容，然后删除冲突文件。

你可以从以下入口管理生成的冲突文件：

- **设置 -> PKV Sync -> 冲突文件**
- **PKV Sync：列出冲突文件**
- **PKV Sync：删除冲突文件**

删除操作只针对 PKV Sync 生成的冲突文件名。诸如 `my.conflict-resolution-notes.md` 这类普通文件仍会被纳入同步。

## 删除远端笔记库

插件设置页面为当前所选远端笔记库提供 **删除** 操作。此操作会：

1. 打开一个确认弹窗，要求你输入完全一致的笔记库名称。
2. 调用服务器的 `DELETE /api/vaults/:id` 接口，删除该笔记库的记录、它的 git 仓库以及该笔记库的 blob 引用。
3. 将本地 Obsidian 笔记库从 PKV Sync 中解除关联（本地磁盘上的文件不会被改动）。

笔记库删除在服务器端不可恢复。如果你需要后悔余地，请先通过 `pkvsyncd backup` 做一次备份。其他当前连接到已删除笔记库的设备会在下次同步时收到 404，需要重新配对到另一个笔记库。

## 设备令牌

登录会签发一个 bearer 设备令牌。已认证的使用会刷新该令牌，闲置 90 天的设备会过期，且单个令牌最长有效 365 天。插件会保留一个稳定的设备 ID，因此在同一设备上再次登录会替换该设备此前的活跃令牌，而不会累积出多个重复项。

Obsidian 插件会将当前的令牌和部署密钥保存在 `<vault>/.obsidian/plugins/pkv-sync/data.json` 中。请将该文件视为敏感数据：保护好明文备份和云同步目标，不要分享出去。如果该文件可能泄露，请退出登录，或请管理员吊销该设备令牌，然后重新连接。

- 在插件设置中可以退出当前设备的登录。
- 请管理员为遗失的设备吊销令牌。
- 修改密码会保持当前设备的登录状态，并吊销你其他设备的令牌。
- 如果管理员禁用你的账号或吊销当前设备令牌，新的 API 请求会失败，已打开的实时事件流也会在 token 复查后关闭。

## MCP 访问

如果你的管理员启用了 MCP（内嵌在主服务 `/mcp`，或通过 `pkvsyncd mcp` 命令运行），AI 工具就可以通过 bearer 设备令牌访问你的笔记库。MCP 提供笔记库列表、文件列表、按 HEAD 或某次 commit 读取文件、简单文本搜索，以及受乐观并发保护的显式写入/删除工具。stdio 与 Streamable HTTP 的配置示例请见 [`mcp-howto.md`](./mcp-howto.md)。

## 命令

PKV Sync 在命令面板中添加了以下命令：

- 显示同步状态
- 刷新账号信息
- 立即手动同步
- 查看同步状态详情
- 检查 PKV Sync 插件更新
- 导入当前笔记库到 PKV Sync（从 Obsidian Sync 一次性迁移）
- 列出冲突文件
- 删除冲突文件
- 解决冲突文件
- 查看文件历史
- 查看仓库历史

## 隐私提醒

PKV Sync 默认并非端到端加密。服务器管理员以及任何拥有服务器文件系统访问权限的人都可以读取已同步的笔记库内容和附件。请仅在你信任的服务器与管理员上使用它。

如果你今天就需要对服务器本身保密，本项目推荐使用 `git-crypt` 作为每个笔记库的过渡性加密层，请参见 [`git-crypt-howto.md`](./git-crypt-howto.md)。原生的每库 E2EE 在 1.x 路线图上，不在 1.0 版本范围之内。
