# PKV Sync 用户手册

[English](./user-manual.md) | 简体中文

本文面向连接已有 PKV Sync 服务端的 Obsidian 用户。开始前，请向服务端管理员获取服务端分享 URL，以及账号或邀请码。

## 手动安装插件

1. 从对应的 GitHub Release 下载 `pkv-sync-plugin.zip`。
2. 解压到你的 Obsidian 仓库：

   ```text
   <vault>/.obsidian/plugins/pkv-sync/
   ```

3. 在 Obsidian 中启用社区插件。
4. 启用 **PKV Sync**。

解压后的目录应包含 `main.js`、`manifest.json` 和 `styles.css`。

## 连接服务端

服务端分享 URL 通常类似：

```text
https://sync.example.com/k_xxx/
```

打开 **设置 -> PKV Sync**，粘贴分享 URL，然后点击 **连接**。如果 URL 中已经包含部署密钥，插件会自动填写。

如果填错服务端，或需要切换到另一个自托管服务端，可以在登录页面点击 **修改服务器** 回到服务端设置，无需重装插件。

## 登录或注册

注册流程取决于服务端运行时设置：

- **禁用注册**：管理员必须先创建账号。
- **仅邀请码**：输入管理员提供的邀请码。
- **开放注册**：可以直接创建账号。

登录后，选择已有远端笔记库或创建新的远端笔记库。如果你把一个本地已经存在且内容与远端完全一致的仓库连接到该远端仓库，PKV Sync 会把匹配文件纳入本地同步索引，而不是生成一整库冲突文件。

## 同步行为

PKV Sync 在 Obsidian 内同步当前仓库：

- 本地文件变化会在短暂防抖后推送。
- 插件会定期轮询远端变化。
- 设置页和命令面板都可以手动同步。
- 相关文件创建、修改、删除事件会安排同步。
- 窗口失焦时可触发同步。
- 启动时会根据仓库内容和本地同步索引识别未同步的本地变化。

上传大附件时，请保持 Obsidian 打开。插件连接后会读取服务端配置，并使用服务端提供的文本扩展名列表和最大文件大小规则。

## 上次同步时间

设置页会用相对时间显示上次成功同步时间。点击旁边的小展开控件可显示精确时间：

```text
YYYY/MM/DD HH:MM:SS
```

插件使用你选择的 IANA 时区，默认是 `Asia/Shanghai`。

## 冲突文件

如果两台设备离线编辑了同一个文件，PKV Sync 会保留两个版本。备用版本会保存为生成的冲突文件：

```text
note.md
note.conflict-2026-04-25-143022-Desktop.md
```

生成的冲突文件不会继续参与同步。请在 Obsidian 中查看两个文件，手动合并需要保留的内容，然后删除冲突文件。

可以通过以下入口管理生成的冲突文件：

- **设置 -> PKV Sync -> 冲突文件**
- **PKV Sync: 列出冲突文件**
- **PKV Sync: 删除冲突文件**

删除动作只会处理 PKV Sync 生成格式的冲突文件。类似 `my.conflict-resolution-notes.md` 的普通文件仍会正常同步。

## 设备 Token

登录会签发一个有效期为 90 天的 bearer 设备 token。插件会保存稳定设备 ID，因此同一设备重新登录时会替换该设备旧的活跃 token，而不是不断累积重复 token。

- 可以在插件设置中退出当前设备。
- 设备丢失后，请让管理员撤销对应设备 token。
- 修改密码会保留当前设备登录状态，并撤销其他设备 token。

## 命令

PKV Sync 会添加以下命令面板动作：

- 显示同步状态
- 刷新账号信息
- 立即手动同步
- 查看同步状态详情
- 列出冲突文件
- 删除冲突文件

## 隐私提醒

PKV Sync 不提供端到端加密。服务端管理员，以及任何拥有服务端文件系统访问权限的人，都可以读取已同步的仓库内容和附件。请只连接你信任的服务端和管理员。
