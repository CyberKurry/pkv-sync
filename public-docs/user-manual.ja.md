# PKV Sync ユーザーマニュアル

[English](./user-manual.md) | [简体中文](./user-manual.zh-CN.md) | [繁體中文](./user-manual.zh-Hant.md) | 日本語 | [한국어](./user-manual.ko.md)

ドキュメントバージョン: v1.4.3。

このマニュアルは、既存の PKV Sync サーバーに接続する Obsidian ユーザー向けです。開始する前に、サーバー管理者からサーバー共有 URL と、アカウントまたは招待コードを入手してください。

## プラグインの手動インストール

1. 対応する GitHub リリースから `pkv-sync-plugin.zip` をダウンロードします。
2. アーカイブを Vault に展開します。

   ```text
   <vault>/.obsidian/plugins/pkv-sync/
   ```

3. Obsidian でコミュニティプラグインを有効にします。
4. **PKV Sync** を有効にします。

展開したディレクトリには `main.js`、`manifest.json`、`styles.css` が含まれている必要があります。

1.3.2 以降、このディレクトリを手動で置き換えてもサインアウトされません。ログインとデバイス ID は、プラグインディレクトリの `data.json` とは別に Obsidian のデバイスローカルストレージに保存されます。

## プラグインの更新

PKV Sync の設定ページには **アップデート** セクションがあります。デフォルトでは、プラグインは接続している PKV Sync サーバーからバンドルされたプラグインのバージョンを確認します。これはセルフホスト環境で推奨されるソースで、サーバーをアップグレードすると対応するプラグインアセットも公開されるためです。サーバーが `public_host` で設定されている場合、プラグインアセットの URL はその外部ホストに固定されます。必要に応じて、アップデートのソースを GitHub リリースに切り替えることもできます。

アップデートが利用可能な場合、**今すぐ更新** をクリックすると、`main.js`、`manifest.json`、および存在する場合は `styles.css` をダウンロードし、SHA-256 ハッシュを検証してプラグインファイルを書き込み、Obsidian の再読み込みを促します。コマンドパレットには **PKV Sync: PKV Sync プラグインのアップデートを確認** も含まれます。

## サーバーへの接続

サーバー共有 URL は通常、次のような形式です。

```text
https://sync.example.com/k_xxx/
```

**設定 -> PKV Sync** を開き、共有 URL を貼り付けて **接続** をクリックします。デプロイメントキーが URL に埋め込まれている場合、プラグインが自動的に入力します。

誤ったサーバーを入力した場合や、別のセルフホストサーバーに移行する必要がある場合は、ログイン画面の **サーバーを変更** を使用して、プラグインを再インストールせずにサーバー設定に戻ることができます。

## ログインまたは登録

登録の動作はサーバーのランタイム設定によって異なります。

- **無効**: 管理者がアカウントを作成する必要があります。
- **招待のみ**: 管理者から提供された招待コードを入力します。
- **オープン**: 直接アカウントを作成できます。

公開登録と招待コード登録は、setup と同じ強力なパスワードポリシーを使用します。12 文字以上で、大文字、小文字、数字を含める必要があります。

ログイン後、既存のリモート Vault を選択するか、新しい Vault を作成します。選択したリモート Vault とすでに完全に同一のローカル Vault を接続すると、PKV Sync は Vault 全体に対する競合ファイルを生成する代わりに、一致するファイルをローカルの同期インデックスに取り込みます。

## 同期の挙動

PKV Sync は Obsidian 内部で動作し、現在の Vault を同期します。

- ローカルファイルの変更は、短いデバウンス間隔の後にプッシュされます。
- リモートの変更は定期的にポーリングされます。
- 設定ページとコマンドパレットから手動同期が可能です。
- 関連するファイルの作成・変更・削除イベントが同期をスケジュールします。
- ウィンドウのフォーカスが外れると同期がトリガーされることがあります。
- 起動時には、Vault の内容とローカル同期インデックスから未同期のローカル変更を検出します。

大きな添付ファイルのアップロード中は Obsidian を開いたままにしてください。プラグインは接続後にサーバー設定を読み込み、サーバーが提供するテキスト拡張子リストと最大ファイルサイズのルールを使用します。

Version 1.2.1 は sync 中の重複した plugin 作業を減らします。pull と push の各段階は可能な場合に同じ scan を再利用し、conditional pull は余分な request を避け、変更ファイルの snapshot は上限付きの並列処理で処理されます。

## `.obsidian` の選択的同期

PKV Sync は、Vault ごとの許可リストを通じて、選択した Obsidian 設定ファイルを同期できます。新しいリモート Vault は、テーマ、CSS スニペット、ホットキー、アプリ設定、外観設定、有効化されたプラグインリストのルールから始まります。

既存のリモート Vault は、オプトインするまで空の許可リストを保持します。**設定 -> PKV Sync** で現在の Vault を選択し、**.obsidian 同期ルール** を編集して保存します。推奨の初期リストボタンは、新しい Vault に使用されるのと同じ初期ルールを入力します。

プラグインのコードとプラグインの設定は、デフォルトでは同期されません。`.obsidian/plugins/**` やプラグインの `data.json` ファイルなどの高度なルールを追加する前に、[`dot-obsidian-sync-howto.md`](./dot-obsidian-sync-howto.md) を参照してください。

## 最終同期時刻

設定ページには、最後に成功した同期が相対時刻で表示されます。隣の小さな展開ボタンを使うと、正確なタイムスタンプを次の形式で表示できます。

```text
YYYY/MM/DD HH:MM:SS
```

プラグインは選択された IANA タイムゾーンを使用し、デフォルトは `Asia/Shanghai` です。

## テーマ

プラグインの設定ページには、3 つの選択肢を持つ **テーマ** ドロップダウンがあります。

- **自動**（デフォルト）: 現在の Obsidian アプリのテーマに従います。
- **ライト** / **ダーク**: アプリのテーマに関係なく、PKV Sync 設定 UI に特定のテーマを強制します。

この設定はプラグイン自身のモーダル（設定、同期ステータス、競合リゾルバー、ファイル/Vault 履歴）にのみ影響し、Obsidian アプリのテーマは変更しません。

## 履歴、差分、復元

サーバーが履歴サポートを報告しており、プラグイン設定で **履歴と差分 UI を有効にする** がオンの場合、次の場所からファイル履歴を確認できます。

- **PKV Sync: ファイル履歴を表示**
- **PKV Sync: Vault 履歴を表示**
- ファイルの右クリックメニュー: **PKV Sync: ファイル履歴**
- ファイルの右クリックメニュー: **PKV Sync: 前のバージョンと比較**

ファイル履歴のモーダルには、現在のファイルの commit が、時刻、デバイス、commit id、変更タイプとともに一覧表示されます。テキストファイルでは unified diff を表示できます。バイナリファイルは一覧表示と復元が可能ですが、PKV Sync はバイナリ差分を描画しません。

現在の同期フィルターで拒否されたパスは、ファイル履歴、Vault 履歴サマリー、diff ビュー、ファイル読み取り画面から隠されます。

Vault 履歴のモーダルには、Vault 全体の最新の commit が、時刻、デバイス、変更数、commit id とともに一覧表示されます。commit を開くと、ファイル単位の変更サマリーを確認できます。

バージョンを復元すると、サーバーから選択した履歴コンテンツが読み出され、ローカルの Obsidian Vault に書き戻され、通常の同期エンジンがその書き込みを新しい commit としてプッシュします。現在のローカルファイルが最後に同期したハッシュと異なる場合、確認ダイアログで未同期のローカル変更が上書きされる旨が警告されます。

PKV Sync はプラグインに完全なオフライン履歴キャッシュを保持しません。履歴と差分ビューにはサーバーへの到達性が必要です。

## 競合ファイル

PKV Sync はプッシュファースト同期プロトコルによる自動マージを採用しています。並行編集の処理方法は重複の種類によって異なります。

- **同じファイル、異なる行**——編集内容は両デバイスで自動的にマージされます。コンフリクトファイルは生成されません。
- **同じファイル、同じ行**——元のファイルの横に `.conflict-*` ファイルが生成され、標準マージマーカー付きの競合内容が含まれます。
- **削除と変更の競合**——ファイルは変更内容を保持したまま存続します。削除側のデバイスには競合通知が届き、再度削除するか残すかを選択できます。

`.conflict-*` ファイルが生成された場合、以降の同期から除外されます。Obsidian で両方のファイルを確認し、残したい内容をマージしてから、競合ファイルを削除してください。

生成された競合ファイルは次の場所から管理できます。

- **設定 -> PKV Sync -> 競合ファイル**
- **PKV Sync: 競合ファイルを一覧表示**
- **PKV Sync: 競合ファイルを削除**

削除アクションは、PKV Sync が生成した競合ファイル名のみを対象とします。`my.conflict-resolution-notes.md` のような通常のファイルは引き続き同期の対象です。

## リモート Vault の削除

プラグインの設定ページには、現在選択中のリモート Vault に対する **Vault を削除** アクションがあります。このアクションは次の動作を行います。

1. 確認モーダルを開き、正確な Vault 名を入力するよう求めます。
2. サーバーの `DELETE /api/vaults/:id` エンドポイントを呼び出し、その Vault の行、git リポジトリ、blob 参照を削除します。
3. ローカルの Obsidian Vault を PKV Sync から切り離します（ローカルファイルはディスク上にそのまま残ります）。

Vault の削除はサーバー上では取り消せません。セーフティネットが必要な場合は、事前に `pkvsyncd backup` でバックアップを取得してください。削除された Vault に現在接続している他のデバイスは、次回の同期で 404 を受け取り、別の Vault に再ペアリングする必要があります。

## デバイストークン

ログインすると bearer デバイストークンが発行されます。認証付きの利用ではトークンが更新されますが、90 日間アイドル状態のデバイスは期限切れになり、各トークンには 365 日の絶対有効期限があります。プラグインは安定したデバイス ID を保持するため、同じデバイスから再度ログインすると、そのデバイスの以前のアクティブトークンが置き換えられ、重複が蓄積されることはありません。

アクティブなトークン、デプロイメントキー、ログイン状態、安定したデバイス ID は Obsidian のデバイスローカルストレージに保存されます。Vault ローカルのプラグイン `data.json` は非機密の設定と同期インデックスだけを保持します。現在のビルドでは同期インデックスの key にデプロイメントキーを含めず、古い機密情報入りのインデックス項目は次回プラグインデータを書き込むときに破棄されます。Obsidian のデバイスローカルストレージ、平文バックアップ、クラウド同期先、古い `data.json` のコピーを保護してください。これらが漏洩した可能性がある場合は、ログアウトするか、管理者にデバイストークンの取り消しを依頼してから、再度接続してください。デプロイメントキーが露出した場合は、デプロイメントキーもローテーションしてください。

- プラグイン設定から現在のデバイスをログアウトできます。
- 紛失したデバイスのトークンの取り消しは管理者に依頼してください。
- パスワードを変更すると、現在のデバイスはサインインしたままで、他のデバイストークンが取り消されます。
- 管理者がアカウントを無効化するか現在のデバイストークンを取り消すと、新しい API リクエストは失敗し、開いているライブイベントストリームも token 再検証後に閉じられます。

## MCP アクセス

管理者が MCP を有効にしている場合（メインサーバーの `/mcp` に埋め込むか、`pkvsyncd mcp` コマンドで実行）、AI ツールは bearer デバイストークンを使用して Vault にアクセスできます。MCP は Vault 一覧、ファイル一覧、HEAD または commit でのファイル読み取り、シンプルなテキスト検索、楽観的並行性制御で保護された明示的な書き込み/削除ツールを提供します。stdio と Streamable HTTP のセットアップ例については [`mcp-howto.md`](./mcp-howto.md) を参照してください。

## コマンド

PKV Sync は次のコマンドパレットアクションを追加します。

- 同期ステータスを表示
- アカウント情報を更新
- 今すぐ手動同期
- 同期ステータスの詳細を表示
- PKV Sync プラグインのアップデートを確認
- 現在の Vault を PKV Sync にインポート（Obsidian Sync からのワンショット移行）
- 競合ファイルを一覧表示
- 競合ファイルを削除
- 競合ファイルを解決
- ファイル履歴を表示
- Vault 履歴を表示

## プライバシーに関する注意

PKV Sync はデフォルトでエンドツーエンド暗号化されていません。サーバー管理者およびサーバーファイルシステムへのアクセス権を持つ者は、同期された Vault の内容と添付ファイルを読むことができます。信頼できるサーバーと管理者の下でのみ使用してください。

現時点でサーバー自体からの機密保持が必要な場合、プロジェクトは暫定的な Vault 単位の暗号化レイヤーとして `git-crypt` を推奨します。詳細は [`git-crypt-howto.md`](./git-crypt-howto.md) を参照してください。ネイティブな Vault 単位の E2EE は 1.x ロードマップに含まれており、1.0 リリースには含まれません。
