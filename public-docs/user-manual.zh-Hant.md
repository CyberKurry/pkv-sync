# PKV Sync 使用者手冊

[English](./user-manual.md) | [简体中文](./user-manual.zh-CN.md) | 繁體中文 | [日本語](./user-manual.ja.md) | [한국어](./user-manual.ko.md)

文件版本：v1.1.1。

本手冊面向連接既有 PKV Sync 伺服器的 Obsidian 使用者。開始之前，請向伺服器管理員取得伺服器分享 URL，以及帳號或邀請碼。

## 手動安裝外掛

1. 從對應的 GitHub Release 下載 `pkv-sync-plugin.zip`。
2. 將壓縮檔解壓到你的筆記庫：

   ```text
   <vault>/.obsidian/plugins/pkv-sync/
   ```

3. 在 Obsidian 中啟用社群外掛。
4. 啟用 **PKV Sync**。

解壓後的目錄應該包含 `main.js`、`manifest.json` 與 `styles.css`。

## 外掛更新

PKV Sync 設定頁包含 **更新** 區塊。預設情況下，外掛會向已連接的 PKV Sync 伺服器查詢隨伺服器打包的外掛版本；對自架部署而言，這是首選來源，因為升級伺服器時也會一併發布相符的外掛資產。當伺服器設定了 `public_host` 時，外掛資產的 URL 會錨定到該外部主機。如有需要，你可以將更新來源切換為 GitHub Release。

當有可用更新時，**立即更新** 會下載 `main.js`、`manifest.json` 與（若存在）`styles.css`，驗證 SHA-256 雜湊，寫入外掛檔案，並提示你重新載入 Obsidian。指令面板中也提供 **PKV Sync：檢查 PKV Sync 外掛更新**。

## 連接伺服器

伺服器分享 URL 通常長這樣：

```text
https://sync.example.com/k_xxx/
```

開啟 **設定 -> PKV Sync**，貼上分享 URL，然後點選 **連線**。如果部署密鑰已嵌入 URL 中，外掛會自動填入。

如果輸入了錯誤的伺服器，或需要切換到另一台自架伺服器，可以在登入畫面使用 **修改伺服器** 回到伺服器設定，而無須重新安裝外掛。

## 登入或註冊

是否能註冊取決於伺服器的執行階段設定：

- **停用**：必須由管理員為你建立帳號。
- **僅邀請**：輸入管理員提供的邀請碼。
- **開放**：可直接建立帳號。

公開註冊和邀請碼註冊都使用與 setup 相同的強密碼策略：至少 12 個字元，並包含大寫字母、小寫字母和數字。

登入後，請選擇既有的遠端筆記庫，或建立新的。當你連接的本機筆記庫已經與所選遠端筆記庫完全一致時，PKV Sync 會把相符的檔案納入本機同步索引，而不會為整個筆記庫產生一堆衝突檔案。

## 同步行為

PKV Sync 在 Obsidian 中執行，並同步目前的筆記庫：

- 本機檔案變更會在短暫去抖延遲後推送。
- 遠端變更會定期輪詢。
- 可從設定頁與指令面板手動同步。
- 相關的檔案建立／修改／刪除事件會排程一次同步。
- 視窗失焦也可能觸發同步。
- 啟動時，會根據筆記庫內容與本機同步索引偵測尚未同步的本機變更。

上傳大型附件時請保持 Obsidian 開啟。外掛在連接後會讀取伺服器設定，並使用伺服器提供的純文字副檔名清單與最大檔案大小規則。

## 選擇性同步 `.obsidian`

PKV Sync 可透過每個筆記庫的允許列表同步選定的 Obsidian 設定檔。新建的遠端筆記庫會預設包含主題、CSS snippet、快捷鍵、應用偏好、外觀偏好以及已啟用外掛清單的規則。

既有的遠端筆記庫會保持空的允許列表，直到你主動加入。請在 **設定 -> PKV Sync** 中選擇目前的筆記庫，編輯 **.obsidian 同步允許列表**，然後儲存。「套用建議起始清單」按鈕會填入與新筆記庫相同的起始規則。

外掛程式碼與外掛設定預設並不同步。新增 `.obsidian/plugins/**` 或外掛 `data.json` 之類的進階規則之前，請參閱 [`dot-obsidian-sync-howto.md`](./dot-obsidian-sync-howto.md)。

## 上次同步時間

設定頁會以相對時間顯示上次成功同步的時間。點旁邊的小展開鈕可以顯示確切時間戳，格式如下：

```text
YYYY/MM/DD HH:MM:SS
```

外掛使用所選的 IANA 時區，預設為 `Asia/Shanghai`。

## 外觀

外掛設定頁有一個 **外觀** 下拉選單，提供三種選項：

- **自動**（預設）：跟隨目前的 Obsidian 應用程式主題。
- **淺色** / **深色**：無論應用程式主題為何，都強制 PKV Sync 設定介面使用指定主題。

此設定只會影響外掛自己的對話框（設定、同步狀態、衝突解決器、檔案／筆記庫歷史）。它不會更動 Obsidian 應用程式主題。

## 歷史、差異與還原

當伺服器回報支援歷史功能，且外掛設定中的 **開啟歷史與差異介面** 已啟用時，你可以從以下入口檢視檔案歷史：

- **PKV Sync：查看檔案歷史**
- **PKV Sync：查看筆記庫歷史**
- 檔案右鍵選單：**PKV Sync：檔案歷史**
- 檔案右鍵選單：**PKV Sync：與上一版對比**

檔案歷史視窗會列出目前檔案的提交，包含時間、裝置、commit id 與變更類型。純文字檔案可顯示 unified diff。二進位檔案可被列出與還原，但 PKV Sync 不會渲染二進位差異。

被目前同步過濾規則拒絕的路徑會從檔案歷史、筆記庫歷史摘要、diff 檢視和檔案讀取介面中隱藏。

筆記庫歷史視窗會列出整個筆記庫最近的提交，包含時間、裝置、變更數量與 commit id。開啟某個提交可以看到逐檔的變更摘要。

還原某個版本時，會從伺服器讀取所選的歷史內容，寫回本機 Obsidian 筆記庫，並讓正常的同步引擎把這次寫入視為新提交推送出去。如果目前的本機檔案與上次同步的雜湊不同，確認對話框會警告你尚未同步的本機變更將被覆寫。

PKV Sync 並不在外掛端保留完整的離線歷史快取。歷史與差異檢視功能要求可連線到伺服器。

## 衝突檔案

如果兩台裝置在離線狀態下編輯同一份檔案，PKV Sync 會保留兩個版本。遠端或本機的替代版本會被另存為自動產生的衝突檔案：

```text
note.md
note.conflict-2026-04-25-143022-Desktop.md
```

自動產生的衝突檔案不會被列入後續同步。請在 Obsidian 中比對兩個檔案，合併你想保留的內容，然後刪除衝突檔案。

你可以從以下位置管理自動產生的衝突檔案：

- **設定 -> PKV Sync -> 衝突檔案**
- **PKV Sync：列出衝突檔案**
- **PKV Sync：刪除衝突檔案**

刪除動作只會針對 PKV Sync 自動產生的衝突檔名。像 `my.conflict-resolution-notes.md` 這類普通檔案仍會納入同步。

## 刪除遠端筆記庫

外掛設定頁針對目前選定的遠端筆記庫提供 **刪除筆記庫** 動作。此動作會：

1. 開啟確認對話框，要求你輸入完整的筆記庫名稱。
2. 呼叫伺服器的 `DELETE /api/vaults/:id` 端點，刪除該筆記庫的資料列、git 儲存庫，以及屬於該筆記庫的 blob 參照。
3. 把本機 Obsidian 筆記庫從 PKV Sync 解除關聯（本機磁碟上的檔案保持原樣）。

筆記庫刪除在伺服器端不可還原。如果需要安全網，請先透過 `pkvsyncd backup` 進行備份。其他目前已連接到該筆記庫的裝置在下一次同步時會收到 404，並需要重新配對到另一個筆記庫。

## 裝置權杖

登入時會核發一個 bearer 裝置權杖。經過認證的使用會更新權杖，閒置 90 天的裝置會逾期，且單個權杖最長有效 365 天。外掛會保留穩定的裝置 ID，因此從同一台裝置再次登入時會替換該裝置先前的活躍權杖，而不是不斷累積重複項目。

Obsidian 外掛會將活躍權杖與部署密鑰儲存於 `<vault>/.obsidian/plugins/pkv-sync/data.json`。請將該檔案視為敏感資料：保護明文備份與雲端同步目標，且不要分享它。如果該檔案可能外洩，請登出，或請管理員撤銷裝置權杖，然後重新連接。

- 使用外掛設定從目前裝置登出。
- 請管理員為遺失的裝置撤銷權杖。
- 變更密碼會保留目前裝置的登入狀態，並撤銷你其他裝置的權杖。
- 如果管理員停用你的帳號或撤銷目前裝置權杖，新的 API 請求會失敗，已開啟的即時事件流也會在 token 重新檢查後關閉。

## MCP 存取

如果你的管理員啟用了 MCP（內嵌在主服務 `/mcp`，或透過 `pkvsyncd mcp` 指令執行），AI 工具可透過 bearer 裝置權杖存取你的筆記庫。MCP 提供筆記庫列舉、檔案列舉、在 HEAD 或指定 commit 讀取檔案、簡易文字搜尋，以及由樂觀並行控制守護的明確寫入／刪除工具。stdio 與 Streamable HTTP 的設定範例請見 [`mcp-howto.md`](./mcp-howto.md)。

## 指令

PKV Sync 在指令面板中新增以下動作：

- 顯示同步狀態
- 重新整理帳號資訊
- 立即手動同步
- 查看同步狀態詳情
- 檢查 PKV Sync 外掛更新
- 匯入目前筆記庫到 PKV Sync（從 Obsidian Sync 一次性遷移）
- 列出衝突檔案
- 刪除衝突檔案
- 解決衝突檔案
- 查看檔案歷史
- 查看筆記庫歷史

## 隱私提醒

PKV Sync 預設並非端對端加密。伺服器管理員以及具有伺服器檔案系統存取權的人都能讀取已同步的筆記庫內容與附件。請只搭配你信任的伺服器與管理員使用。

如果你目前就需要對伺服器本身保密，本專案建議使用 `git-crypt` 作為過渡性的每個筆記庫加密層；詳見 [`git-crypt-howto.md`](./git-crypt-howto.md)。原生的每個筆記庫 E2EE 規劃在 1.x 路線圖中，並不包含於 1.0 版本內。
